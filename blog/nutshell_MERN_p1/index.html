THE MERN STACK

MERN stands for MongoDB/Express/React/Node.js and is a slight alternative to the commonly used MEAN stack (which uses Angular instead of React). These four tools are essential when it comes to building any web based application.

  • MongoDB: a NoSQL database that uses JSON-like documents. This kicks some serious ass with javascript since it goes so well with JSON. There are a ton of awesome advantages to using a NoSQL database and you can read more about them on the MongoDB website.
  • Express: a popular Node package that makes things easy - used for flexible routing. Comes with loads of features such as middleware frameworks and templating engines. Don't worry - we will cover this in more detail later. Read more on the Express website.
  • React: TBD. Read more on the React website.
  • Node.js: the heart and soul of our web application. It uses Google's V8 engine and this essentially allows us to run Javascript on a computer (outside of a web browser). We can do things like create a server, read/update/delete files, and easily communicate with a database. POWER OVERWHELMING. Read more on the Node.js website.

MODEL-VIEW-CONTROLLER

The final thing to know before building a web application is the model-view-controller (MVC) design pattern. As the name suggests, it consists of three interconnected parts:

  • Model: the data manager; define data structures/methods to handle our stored data
  • View: the output; display/organize information for the user.
  • Controller: the bridge; links the model and view by getting requests, pulling data from model, passing it all back to view.
